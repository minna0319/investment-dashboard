1. 基本面

公司簡介

Cloudflare 是一家總部在美國舊金山的雲端基礎設施公司，用一句話說就是：幫網站和 App 跑得更快、更安全的「網路守門員」。

全球大約 21.3% 的網站都在使用 Cloudflare 的服務（2026 年 1 月數據，W3Techs）。

3. 估值面

股票類型判定

Cloudflare 屬於 TYPE_C（美股科技成長股） — 高成長、高估值的 SaaS/雲端公司。

關鍵估值指標

指標數值解讀
Forward P/E（預估本益比）~185x（$213 ÷ $1.15）非常高，但成長股常見
P/S（股價營收比）~34x歷史中位數 23x，目前偏高
P/S 歷史區間11.6x ~ 110.6x目前在偏高的 60-70% 區間
P/S ratio（股價營收比） → 市場願意用幾倍的營收來買這家公司。Cloudflare 的 34 倍表示市場願意用 34 年的營收來買它，通常高成長公司才有這種待遇。

雙軌估值計算

方法 A（歷史 PS 區間法）

由於 Cloudflare GAAP EPS 仍為負，PE 法失真，改用 PS ratio。

```

合理 PS = 歷史中位數 23x

2025 每股營收 = $21.68 億 ÷ 3.55 億股 ≈ $6.10

fair_value_a = 23 × $6.10 = $140.3

```

方法 B（PEG 估值法 — 用 Non-GAAP Forward EPS）

```

2026E Non-GAAP EPS = $1.15

EPS 成長率 ≈ 19%（從 $0.97 到 $1.15）

合理 PE = 19 × 1（PEG=1） = 19x

但 TYPE_C 科技股 PE 上限 40x，且高成長公司 PEG=1 太保守

調整：取 PEG=1.5 的合理水準 → PE = 19 × 1.5 = 28.5x

再考量營收成長 29% → 合理 PE 取 30x

fair_value_b = 30 × $1.15 = $34.5 ← 此值過低，因 Non-GAAP EPS 不含股票獎酬

⚠️ PEG 法在 GAAP 虧損的高成長公司容易失真

改用 Forward PS 法作為方法 B：

2026E 營收 = $27.9 億 ÷ 3.77 億股 ≈ $7.40/股

合理 Forward PS = 20x（考慮 29% 成長 + 行業水準保守取值）

fair_value_b = 20 × $7.40 = $148.0

```

差距判定

```

差距 = |$140.3 - $148.0| / $140.3 = 5.5%

差距 ≤ 20% → 取平均

anchor = ($140.3 + $148.0) / 2 = $144.2

```

進場區間（TYPE_C：anchor × 0.82 ~ 0.88）

```

entry_low = $144.2 × 0.82 ≈ $118

entry_high = $144.2 × 0.88 ≈ $127

```

估值結論

項目數值
方法 A（歷史 PS 區間法）$140（PS 23x × 每股營收 $6.10）
方法 B（Forward PS 法）$148（Forward PS 20x × 2026E 每股營收 $7.40）
差距5.5% → 取平均
錨定價$144

現價 $213 大幅高於合理進場區間，目前追買風險偏高。 如果用分析師的平均目標價 $235-$245 來看，市場顯然給了很高的成長溢價（premium），但以價值投資的角度，這個價位不適合新手入場。
