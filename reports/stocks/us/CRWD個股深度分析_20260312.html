1. 基本面

公司一句話介紹

CrowdStrike 是全球最大的「純資安」公司，靠一個叫 Falcon 的雲端平台，幫企業擋駭客、抓病毒、保護電腦和雲端伺服器。

主要業務

  • 端點安全（Endpoint Security） → 保護企業裡每一台電腦、筆電、伺服器不被駭客入侵。就像每台電腦都有一個 AI 保鏢
  • XDR（延伸偵測與回應） → 不只看電腦，還把網路、雲端、手機的資安事件通通串起來一起分析，讓駭客無處躲
  • 雲端安全（Cloud Security / CNAPP） → 企業搬到雲端後，保護他們在 AWS、Azure、GCP 上的資料安全
  • 身份安全（Identity Protection） → 防止駭客偷帳號密碼來入侵
  • 威脅情報（Threat Intelligence） → 研究全球駭客組織的行為模式，提前預警

產業鏈定位

```

駭客/威脅 → [CrowdStrike Falcon 平台] → 保護企業端點/雲端/身份

↑

雲端基礎設施（AWS/Azure/GCP）

↑

企業客戶（Fortune 500、政府機關）

```

CrowdStrike 位於資安產業鏈的中游，直接面對企業客戶，上游依賴雲端基礎設施。

競爭優勢（護城河）🏰

  1. 平台鎖定效應（強）：客戶從 CrowdStrike 換到競爭對手的成本很高——要重新部署、重新訓練員工、還可能有資安空窗期
  2. 數據飛輪（強）：2.9 萬客戶的資安數據不斷餵給 AI，讓 Falcon 越用越聰明，新進者很難追上
  3. 品牌信任（中→強）：端點安全市占率 22.5%，是純資安公司中的第一名
  4. 模組擴展（強）：客戶一開始買 1-2 個模組，後來慢慢加到 5-8 個，平均每客戶採用模組數持續增加

管理層

  • FY2026 業績承諾 vs 實際：指引持續上調，全年營收和 EPS 都超過原始指引
  • 2024 年全球當機事件後的危機處理獲得肯定，客戶留存率維持 97%

3. 估值面

股票類型判定

  • 非 ETF → 非 TYPE_E
  • EPS > 0 → 非 TYPE_D
  • 美股科技/SaaS → TYPE_C（美股科技股）

方法 A：歷史 PE 區間法（回顧型）

CrowdStrike 的 GAAP PE 長期為負（因 SBC 高導致 GAAP 虧損），需使用 Non-GAAP PE。

參數數值來源
Forward EPS（FY2027）$4.84（中位數）公司指引
當前 Forward PE~91x（$442/$4.84）計算
歷史 Non-GAAP Forward PE 範圍60x - 120x近 3 年區間
PE 低標（25th）~65x估算
PE 中位數~85x估算

```

fair_value_a = Forward EPS $4.84 × 歷史 PE 中位數 85x = $411

```

⚠️ CrowdStrike 的 PE 即使在「低標」都很高（65x），這是高成長 SaaS 的特性。但因 TYPE_C 上限為 40x 時需改用 PS，而 CRWD 歷史 PE 一直遠高於 40x，此處使用其自身歷史區間更合理，但標註「估值分歧大」。

方法 B：PEG 估值法（前瞻型）

參數數值來源
Forward EPS$4.84FY2027 指引中位數
EPS 年成長率~30%（FY2027 指引成長）公司指引
合理 PEG1（合理定價）標準
合理 PE = 成長率 × 130x計算
TYPE_C PE 上限40x → 使用 30xSOP

```

fair_value_b = Forward EPS $4.84 × 合理 PE 30x = $145

```

⚠️ PEG 法算出的合理價遠低於市價，這反映了市場給予 CRWD 極高的成長溢價。PEG=1 的框架在高溢價 SaaS 股上容易失真。

差距判定

```

差距 = |$411 - $145| / min($411, $145) = $266 / $145 = 183%

→ > 40%：取較低者 + ⚠️ 估值分歧大

anchor = $145（方法 B）

```

但因 PEG=1 的估值框架在此類超高成長 SaaS 上明顯偏保守（整個資安 SaaS 板塊都交易在 PEG 2-3），補充參考：

補充估值：PS ratio 法

參數數值
FY2027 預估營收~$59 億
總市值~$1,090 億
當前 PS~18.5x
同業 PS 中位數PANW ~16x、S ~9x
合理 PS 範圍15-20x（考慮龍頭溢價）
PS 15x 合理市值$885 億 → 每股 ~$360
PS 18x 合理市值$1,062 億 → 每股 ~$431

進場區間（TYPE_C）

考量到 PEG 法嚴重低估此類股票，改用歷史 PE 法（$411）與 PS 法中位（$395）做錨定：

```

anchor = ($411 + $395) / 2 = $403（取兩法平均）

entry_high = $403 × 0.88 = $355

entry_low = $403 × 0.82 = $330

```

技術面微調

  • MA200 = $468.95，離進場區較遠
  • MA50 = $430.76，現價在其上方
  • 前波低點 52 週低 = $321.63
  • entry_low $330 接近 52 週低點，作為強支撐有參考價值

估值結論

```

方法 A（歷史PE區間法）：合理價 $411（Forward EPS $4.84 × 歷史PE中位數 85x）

方法 B（PEG估值法）：合理價 $145（Forward EPS $4.84 × 合理PE 30x, PEG=1）

差距：183% → 取較低者 + ⚠️ 估值分歧大

補充：PS 營收比法合理價 $360-$431

錨定價：$403（歷史PE法 + PS法平均，PEG法因框架限制不適用此類股）

進場區間：$330 ~ $355

備註：⚠️ 估值分歧大（PEG 框架在高溢價 SaaS 失真）。現價 $442 高於進場區間上緣約 24%，建議等回調。

```

💡 白話說：以目前 $442 的價格來看，算是偏貴。理想的進場價大約在 $330-$355 之間，也就是需要再跌 20-25% 才到「甜蜜區」。
